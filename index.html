📋 OSCP Exam Tips
Strategy, methodology, and time management for the OSCP exam
📌 OSCP Format (2024+)6 machines: 3 standalone + 1 AD set (DC + 2 clients). Max 100 pts: 20 per standalone, 40 for full AD chain. Need 70 to pass. 24hr exam + 24hr report.

Time Management

TimeAction
0:00Start AD set — highest value (40 pts), chain required
0:00–4:00Enumerate all machines simultaneously with nmap
4:00–8:00Foothold on AD + 1-2 standalones
8:00–16:00PrivEsc on secured machines, lateral AD movement
16:00–20:00Complete AD chain to DC, finish remaining standalones
20:00–24:00Document everything, screenshots, proof.txt

Must-Do On Every Machine

  • Screenshot: hostname, whoami, ipconfig/ifconfig
  • Screenshot: local.txt / proof.txt content WITH hostname
  • Note all open ports and services found
  • Document exact commands used for exploitation
  • Note credentials found — try on other machines
  • Check for password reuse across all targets

Enumeration Order

bash
# 1. Fast full port scan (run first, let it run in background)
nmap -p- --min-rate 5000 -oA fullscan TARGET &

# 2. Service version scan on found ports
nmap -sV -sC -p PORT1,PORT2 TARGET -oA services

# 3. Web (if 80/443/8080)
gobuster dir -u http://TARGET -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -x php,txt,html &
nikto -h http://TARGET

# 4. SMB (if 445)
enum4linux -a TARGET; smbmap -H TARGET

# 5. searchsploit for detected versions
searchsploit "Apache 2.4.49"

When You're Stuck

💡 Unstuck Strategy 1. Re-enumerate — you missed something. Run gobuster with different wordlists. 2. Check version numbers more carefully — searchsploit every service. 3. Read all source code, comments, and config files found. 4. Try harder on password reuse — spray found creds everywhere. 5. Move to another machine and come back fresh. 6. Check for non-standard ports (8443, 8888, 3000, etc.).
